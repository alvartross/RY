# 에이전트 작업 규칙

이 프로젝트에서 AI 에이전트(Claude 등)와 협업할 때 따라야 할 지침입니다.

## 언어 / 커뮤니케이션

- 모든 응답은 **한국어**로 작성할 것 (코드 내부의 변수명/함수명은 영어 허용)
- 커밋 메시지도 한국어로 작성 (예: `feat: 단어 카드에 발음 버튼 추가`)
- 사용자는 **프로그래밍 학습 중**인 비개발자. 새 개념이 나오면 1~2문장으로 쉽게 설명하고 진행
- **모든 안내는 초등학생도 따라할 수 있게** 최대한 쉽고 자세하게 설명할 것
  - "어디서 뭘 클릭하는지"까지 구체적으로
  - 전문 용어보다 화면에 보이는 글자 그대로 안내
  - 스크린샷 기반이면 "빨간 버튼", "왼쪽 메뉴" 식으로 위치 묘사
  - 코드 붙여넣기가 필요하면 "이것만 복사 → 저기에 붙여넣기 → Run 클릭" 3단계로

## 프로젝트 개요

- **이름**: 영유(영어유치원) 복습용 아이 교육 앱
- **대상**: 영유를 다니는 어린이
- **기능**: 오늘의 단어 학습 카드, 알파벳 따라쓰기 보드
- **스택**: Next.js 16 + React 19 + Tailwind CSS 4 + TypeScript
- **개발 포트**: `8000` 고정 (3000번은 다른 프로젝트 사용 중)

## 코드 스타일

- **과도한 추상화 금지** — 당장 필요한 만큼만 작성. "미래를 위한" 헬퍼/유틸 미리 만들지 말 것
- **주석 최소화** — 변수/함수 이름으로 설명되지 않는 "왜"만 주석으로 남김. "무엇을 하는지"는 이름으로 표현
- 에러 처리는 **경계(사용자 입력, 외부 API)에만**. 내부 함수 간에는 불필요한 방어 코드 금지
- 타입 오류/린트 경고는 반드시 해결 후 작업 완료 보고

## 작업 흐름

1. 복잡한 작업 전엔 접근 방식을 **먼저 설명**하고 사용자 동의 받기
2. UI 변경 시 브라우저에서 실제 동작을 확인한 뒤 완료 보고 (빌드 성공만으로 완료 처리 금지)
3. 완료 보고 시 변경된 파일과 라인 번호를 클릭 가능한 링크로 명시: `[파일명:라인](경로#L라인)`

## 사용자 데이터 안전 (최우선)

localStorage·DB·파일 등 사용자가 입력한 데이터를 다루는 코드는 **항상 보수적으로** 작성한다.

- **동기화/마이그레이션 코드 활성화 직전엔 사전 백업 안내**
  - "📤 내보내기 먼저 받아두세요"를 사용자에게 명시적으로 권유
- **빈 원격 → 로컬 보존 원칙**
  - 클라우드/서버에서 가져온 데이터가 비어있을 때는 **절대 로컬을 덮어쓰지 않음**
  - 양방향 머지는 `updated_at` 기준 최신 우선
- **위험 코드 단계별 출시**
  - 동기화는 한 번에 자동 발동하지 말고, "수동 ⬆ 업로드 먼저 → 그 다음 자동 풀" 순서로
- **destructive 호출 명시 의도 외 금지**
  - `localStorage.setItem(KEY, JSON.stringify({}))` 같은 패턴은 의도 명확할 때만
  - `truncate`, `delete *`, `rm -rf` 등 파괴적 명령은 사용자 명시 허락 후에만
- **사용자 입력 데이터를 다루는 PR은 다음 체크리스트 통과 후 보고**
  - [ ] 빈 원격 케이스 처리 (로컬 보존)
  - [ ] 충돌 해결 정책 명확 (last-write-wins / merge / etc)
  - [ ] 테스트 데이터로 먼저 검증
  - [ ] 사전 백업 안내 메시지

## Git 규칙

- **`git commit`과 `git push`는 사용자가 명시적으로 요청했을 때만** 실행할 것
  - 코드 작업 끝났다고 자동으로 커밋·푸시 금지
  - "커밋해줘", "푸시해줘", "올려줘" 같은 명시 지시가 있어야 함
  - 작업 완료 보고만 하고 사용자 다음 지시를 기다릴 것
- 사용자는 git 초보. 명령 실행 전 **무엇을/왜 하는지 짧게 설명**하고 실행
- 커밋 전 반드시 `git status`와 `git diff`로 변경 내역 확인
- 커밋 메시지 포맷: `<타입>: <설명>`
  - `feat`: 새 기능 / `fix`: 버그 수정 / `docs`: 문서 / `refactor`: 구조 개선 / `style`: 포맷/공백 / `chore`: 설정/빌드
- **절대 금지 (사용자 명시 허락 없이)**
  - `git push --force` / `git reset --hard` / `git clean -fd` 등 되돌릴 수 없는 명령
  - `--no-verify`로 훅 우회
  - 리모트 `main` 브랜치에 force push

## 하지 말 것

- 사용자가 요청하지 않은 기능 추가, 리팩터링, 문서 생성
- 이모지 사용 (명시 요청이 없는 한)
- 할 일 요약을 매 응답 끝에 붙이는 장황한 답변
- `.env`, 비밀키, 개인정보가 들어간 파일 커밋
